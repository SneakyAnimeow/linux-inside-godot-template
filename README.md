# Real Hacker Evolution Boilerplate

Welcome to the **Real Hacker Evolution** boilerplate—a starting template for building 3D hacking simulations in Godot 4. This template demonstrates how to embed a real, fully-functional Alpine Linux Virtual Machine directly into a Godot project using an embedded Node.js runtime and `v86`. 

Instead of relying on simulated or "fake" terminal mechanics, this boilerplate provides a foundation where in-game terminals connect to a true sandboxed operating system.

## 🚀 Key Features

* **Embedded Node.js Runtime:** Uses `Microsoft.JavaScript.NodeApi` to run a fully functional Node.js instance inside a background C# thread native to Godot.
* **True Virtualization:** Runs [v86](https://github.com/copy/v86) (an x86 emulator in WebAssembly) within the embedded Node environment to boot a real `Alpine Linux` ISO.
* **Custom 3D VT100 Terminal:** A high-performance, custom-built VT100 terminal emulator written entirely in C#. It intercepts raw ANSI/VT100 escape sequences from the VM's serial output and paints them onto an in-game Godot `Label` floating on a 3D monitor.
* **Flawless Keyboard Interop:** Full keyboard capturing, sending exact physical inputs (including Ctrl shortcuts like `Ctrl+C` / `Ctrl+D` and directional arrow keys) straight to the Linux Kernel.

## 📂 Project Architecture

```
│       └── images/          # alpine.iso (The Linux OS image)
```

## 🛠️ Build Requirements

1. **Godot 4.3+ (.NET Version)**
2. **.NET 8.0 SDK**
3. **[Bun](https://bun.sh/)** (Used to lightning-fast bundle the TypeScript Node module)

## 🏗️ How Building Works

You do not need to manually bundle the JavaScript payload. The `RealHackerEvolution.csproj` has been configured with an MSBuild `BeforeBuild` target. 
Every time you build the game via Godot or `dotnet build`:
1. `bun install` and `bun run build` are triggered automatically.
2. The compiled `index.cjs`, `v86.wasm`, firmware, and `alpine.iso` are automatically preserved and copied into the correct `AppContext.BaseDirectory/js` directory alongside the game executable.

## 🎮 Controls

* Walk up to the in-game terminal monitor.
* **Enter Terminal:** Press `E` when prompted.
* **Exit Terminal:** Press `Shift + Escape`. (Note: Pressing just `Escape` sends the standard `ESC` character to the VM, allowing you to exit insert mode in Vim).
* Use standard bash commands. All VT100 formatting, colors, and layout instructions are parsed seamlessly.

## 🧠 Under the Hood: The Terminal Interop

When the Godot Autoload (`Program.cs`) starts, it dynamically loads the local `libnode` runtime and opens an event loop. It imports the bundled `index.cjs` file, booting the v86 machine. 

Communication happens via pure asynchronous cross-thread function references:
* C# holds a strong `JSReference` to the `send(char)` function exposed by JS.
* JS holds a callback reference to `onOutput(char)`, invoking it every time the Linux kernel writes to the serial port.
* The `ScreenInteract.cs` script queues these characters, funnels them through a custom VT100 state machine to strip/apply ANSI sequences, and manipulates an in-memory 80x25 character grid before rendering it to a Godot Label.
